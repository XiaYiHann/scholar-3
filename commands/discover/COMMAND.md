进入 `/discover`：维护一个单一 `research/idea.md`（可冻结），并初始化/更新 `research/evidence-ledger.md`。

Discover v2 内部闸门：
1) Intake（状态与约束）
2) Diverge（至少 3 个候选方向）
3) Challenge（反证/风险/失败信号）
4) Converge（收敛到 H/F/C + protocol）
5) Handoff（为 `/proposal` 生成 capsule）

输出（在你的项目中）：
- `research/idea.md`
- `research/idea.meta.yaml`
- `research/evidence-ledger.md`

规则：
- 不编造引用、不编造结果
- 冻结后不允许直接改 idea（只能发起 amendment）
- 建议使用结构化问诊 ID：`discover_start` / `discover_goal` / `discover_constraints` / `discover_diverge_lens` / `discover_freeze_decision`

下一步：
- `/proposal` 创建分阶段 proposal（plan + tasks）
